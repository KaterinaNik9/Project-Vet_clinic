# Project-Vet_clinic
Этот репозиторий содержит код с программой, которая моделирует работу ветеринарной клиники: учет животных, их медицинских записей, назначений (кормление, лечение) и процедур.

1. Цель разработки
Создать консольное приложение на C++, демонстрирующее основные принципы объектно-ориентированного программирования (ООП):
---- инкапсуляцию
---- наследование
---- полиморфизм
---- абстракцию
   
3. Общая архитектура
Проект строится на иерархии классов с использованием абстрактного базового класса Animal и последующим наследованием. Все данные хранятся в контейнере AnimalHospital, который управляет списком животных, их записями и историей болезней.

Основные компоненты:

Иерархия животных (Animal → конкретные виды)
Абстрактный класс Animal
Производные классы: Dog, Cat, Bird, Reptile, ExoticAnimal
Медицинские записи (MedicalRecord)
Композиция с Animal (каждое животное содержит объект MedicalRecord)
Лечебные процедуры (Treatment)
Абстрактный класс Treatment
Производные: Vaccination, Surgery, Examination, Medication
Запись на прием (Appointment)
Связывает животное, врача, дату и тип процедуры
Контейнер (AnimalHospital)
Хранит коллекцию животных (динамический массив указателей на Animal)
Управляет записями и поиском

3. Основной функционал
3.1. Управление животными
Добавление нового животного (динамическое создание через new)

Удаление животного по ID или имени

Просмотр списка всех животных

Поиск животных по критериям:

кличка

вид (порода)

дата поступления

3.2. Медицинская история
Добавление записи в историю болезни (диагноз, дата, лечение)

Просмотр истории конкретного животного

3.3. Назначения и процедуры
Запись животного на прием (Appointment)

Назначение лечения (через полиморфный вызов Treatment)

Вывод расписания кормления и протокола лечения (чисто виртуальные функции)

3.4. Отчеты
Вывод всех животных с их текущим статусом

Статистика по видам животных

Список предстоящих процедур

4. Требования к реализации
4.1. Обязательные требования
Использование абстрактного класса Animal с чисто виртуальными методами:

virtual std::string getFeedSchedule() const = 0;

virtual std::string getTreatmentProtocol() const = 0;

Наследование не менее 5 конкретных классов животных.

Композиция: каждое животное содержит объект MedicalRecord.

Динамическое создание объектов через new и хранение в контейнере по базовому указателю.

Полиморфный вызов методов лечения через указатель на Treatment.

Поиск животных по нескольким критериям (имя, вид, дата).

4.2. Дополнительные требования
Использование std::string для текстовых данных.

Работа с динамической памятью без утечек (деструкторы, правило трёх/пяти).

Консольный интерфейс с меню для взаимодействия с пользователем.

5. Сценарий демонстрации
Запуск программы — приветствие и главное меню.

Добавление животных:

Добавить собаку (кличка "Бобик", порода "Овчарка", дата поступления 2025-03-17)

Добавить кошку (кличка "Мурка", порода "Сиамская", дата поступления 2025-03-18)

Добавить попугая (кличка "Кеша", вид "Ара", дата поступления 2025-03-19)

Просмотр списка животных — вывод всех добавленных.

Добавление медицинской записи:

Для Бобика: диагноз "Отит", назначено лечение "Антибиотики"

Для Мурки: вакцинация от бешенства

Назначение процедур:

Записать Кешу на осмотр 2025-03-20

Поиск:

Найти всех кошек

Найти животных, поступивших после 2025-03-18

Вывод расписания — для каждого животного показать график кормления и протокол лечения.

Удаление животного — например, Бобика (имитация выписки).

Завершение программы — корректное освобождение памяти.
